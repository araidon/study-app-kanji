# 小学1年生向け漢字熟語ゲーム 要件定義書

## 1. 概要

小学1年生で学ぶ漢字80字を使って2字熟語を作るブラウザゲーム。
カードゲーム形式で、制限時間内にできるだけ多くの熟語を作ることを目指す。

## 2. ターゲット

| 項目 | 内容 |
|------|------|
| 対象ユーザー | 小学1年生（およびその保護者） |
| プラットフォーム | iPhone（ブラウザ） |
| 想定利用シーン | 漢字学習、隙間時間の学習ゲーム |

## 3. 画面構成

### 3.1 タイトル画面
- ゲームタイトル表示
- 「ゲーム開始」ボタン

### 3.2 ゲーム画面
- **手札エリア**: 10枚のカードを表示
- **情報表示エリア**:
  - 残り時間（3分からカウントダウン）
  - 現在の得点
  - 残りカード枚数（山札）
- **ゴミ箱**: 不要なカードを捨てる領域

### 3.3 結果画面
- 最終得点
- 作成した熟語一覧（得点内訳付き）
- 「もう一度遊ぶ」ボタン

## 4. ゲームフロー

```
[タイトル画面]
    ↓ ゲーム開始ボタン
[ゲーム画面]
    ↓ 初期化: 80枚シャッフル → 10枚配布
    ↓
    ループ開始
    │
    ├─→ カード1枚目をタッチ（選択状態になる）
    │     │
    │     ├─→ 同じカードをタッチ → 選択解除
    │     ├─→ ゴミ箱をタッチ → カード捨てる → 1枚補充
    │     └─→ 別のカード（2枚目）をタッチ
    │           │
    │           ├─→ 熟語として正解 → 得点加算 → 2枚消費 → 2枚補充
    │           └─→ 熟語として不正解 → 選択解除（ペナルティなし）
    │
    └─← 繰り返し
    ↓
    終了条件: 山札が0枚 または 制限時間3分経過
    ↓
[結果画面]
```

## 5. 機能要件

### 5.1 カード機能

| 機能 | 詳細 |
|------|------|
| 初期配布 | ゲーム開始時に10枚配布 |
| 選択 | タッチで選択状態にする |
| 選択解除 | 選択中のカードを再タッチで解除 |
| 熟語判定 | 2枚選択時に熟語かどうか判定（順番考慮） |
| 補充 | 熟語成立時2枚、カード捨て時1枚補充 |
| 捨てる | 選択中のカードをゴミ箱にタッチで捨てる |

### 5.2 カード捨て機能

- 選択中のカードをゴミ箱にドラッグ/タッチで捨てる
- 捨てたカードは山札から1枚補充
- 捨てたカードは再登場しない（ゲームから除外）
- 捨てる枚数に制限なし

### 5.3 終了条件

| 条件 | 動作 |
|------|------|
| 山札が0枚 | 即座に結果画面へ |
| 制限時間3分経過 | 即座に結果画面へ |

※手札に残ったカードは未使用として扱う（得点対象外）

## 6. 得点システム

### 6.1 基本点

| 熟語タイプ | 点数 | 例 |
|------------|------|-----|
| 苗字 | 1点 | 山田、田中 |
| 一般的な熟語 | 2点 | 火山、天気 |

### 6.2 ボーナス点

| ボーナス条件 | 追加点数 | 例 |
|--------------|----------|-----|
| 順番逆でも意味が通じる | +2点 | 上下/下上 |

### 6.3 対象外

- 一般に使われない熟語はリストに含めない

## 7. UI/UX要件

### 7.1 視覚的フィードバック

| 状態 | 表示 |
|------|------|
| カード選択中 | 選択されていることが分かる視覚効果 |
| 熟語正解時 | 正解を示すアニメーション/エフェクト |
| 熟語不正解時 | 不正解を示す視覚効果 |

### 7.2 音声

- 音声・効果音は不要

### 7.3 レスポンシブ対応

- iPhone（Safari）での表示を最適化
- タッチ操作に対応

## 8. データ要件

### 8.1 漢字データ（80字）

小学1年生で学ぶ教育漢字80字を使用。

### 8.2 熟語データ

以下の情報を持つ熟語リストを事前に用意:

| フィールド | 説明 |
|------------|------|
| 熟語 | 2文字の熟語（例: 火山） |
| タイプ | 苗字 / 一般 |
| 逆順熟語 | 逆順で意味が通じる場合、その熟語 |

## 9. 非機能要件

### 9.1 パフォーマンス

- ゲーム中の操作は即座に反応すること
- タイマーは正確に動作すること

### 9.2 ブラウザ対応

- iPhone Safari で動作すること
- オフラインでも動作することが望ましい（PWA化は将来検討）
